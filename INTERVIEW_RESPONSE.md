# 同弈 (SyncGo) - 面试项目介绍

## 项目概述

**同弈 (SyncGo)** 是一款创新性的同步围棋变体游戏，我作为独立开发者从零构建的商业化桌面应用。项目采用 **Node.js + Electron** 技术栈，实现了完整的跨平台桌面应用开发、复杂游戏逻辑引擎、网络通信和AI算法。

## 核心技术亮点

### 1. 同步落子机制 (核心创新)
- 双方每一步**同时选择落子位置**，提交后才揭示对方走法
- 实现**红色棋子**机制：双方落子重合时产生红棋（永久固定、不可吃、减少气）
- 开发了完整的**同步回合解决算法**，处理碰撞检测、气计算、提子同步等复杂逻辑

### 2. 游戏引擎架构
```
GameEngine (游戏引擎核心)
├── Board (棋盘数据结构 - 19x19数组)
├── Rules (规则验证 - 气计算、打吃判断、劫争)
├── Scoring (中国规则数子法)
└── TimerManager (计时器系统 - 基本用时+3次30秒读秒)
```

### 3. 网络通信架构
- **WebSocket (ws)** 实现局域网对弈的房间发现和消息同步
- **WebRTC P2P** 实现点对点直连通信，降低延迟
- **服务器权威模式**：主机运行游戏服务器，确保游戏状态一致性

### 4. AI引擎实现
- **蒙特卡洛树搜索 (MCTS)** 简化版算法
- 三档难度控制（简单/中等/困难），通过调整模拟次数实现
- 包含启发式评估：中心优先、星位加分、吃子优先、打入/逃孤判断

### 5. 桌面应用开发
- **Electron** 多进程架构（主进程+渲染进程）
- **IPC通信** 实现进程间消息传递
- **Canvas** 高性能棋盘渲染
- **electron-builder** 多平台打包（macOS/Windows/Linux）

## 技术细节

### 核心算法复杂度
- **气计算**：O(n) 邻接搜索，n为棋子数
- **提子检测**：双向BFS遍历棋组
- **打劫判断**：位置哈希集合实现**全局同型检测 (Superko)**

### 网络协议设计
```javascript
// JSON消息协议
{ type: 'submit_move', x, y }      // 提交落子
{ type: 'turn_result', board, ...} // 回合结果
{ type: 'time_update', timers }   // 计时同步
```

### 计时器状态机
```
基本用时 → 读秒1(30s) → 读秒2(30s) → 读秒3(30s) → 超时禁着
```
